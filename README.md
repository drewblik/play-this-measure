# Play This Measure

A camera-first PWA piano teacher organized around the pieces you're learning.
Photograph a page to start a song; photograph a measure that stumps you and the
app reads it, lets you confirm the reading, and teaches it back through a
synchronized play-along.

See `docs/` for the authoritative **TDD** (technical), the **FDD** (UX), the
vision, and the exec summary. `tie-rhythm.html` is the debugged prototype of the
rendering + audio engine — it is **ported** into `src/engine/`, not rewritten.

## Stack

Vite + vanilla JS (ES modules), hand-rolled SVG notation + Web Audio, IndexedDB
(`idb`), a single Vercel serverless proxy for the Anthropic Messages API, hosted
on Vercel as a static PWA. No UI framework, no notation library. (TDD §2.)

## Run

```bash
npm install
npm run dev        # vite dev server (UI only; the /api proxy runs on Vercel)
npm run build      # -> dist/
npm run preview    # serve the production build locally
npm run gen:icons  # regenerate placeholder PWA icons
```

The `/api/claude.js` proxy only runs in a Vercel environment (it needs
`ANTHROPIC_API_KEY`). To exercise it locally use `vercel dev` (from M2 on).

## Environment variables (set in Vercel)

| Var | Used by | Notes |
|---|---|---|
| `ANTHROPIC_API_KEY` | `/api/claude.js` only | The only secret v1 code uses. Never shipped to the client. |
| `DATABASE_URL` | **nothing in v1** | Neon Postgres connection string. Provisioned but **dormant** — v1 never touches it. Reserved landing zone for v2 cloud sync. |

## Development loop

Drive Claude Code (including from the mobile app) → push to `main` → Vercel
deploys → open on the phone. Each branch gets its own Vercel **preview**
deployment.

> **Real data lives on the production URL only.** IndexedDB is per-origin, so
> every preview URL has its own separate on-device storage. Use preview URLs for
> feature testing; use the production URL for your actual songs and lessons.

The in-app build marker (bottom of the home screen) confirms a fresh deploy
reached the device after a push.

## Notes

- `public/sw.js` (not `src/sw.js` as sketched in TDD §2): the service worker
  must be served from the site root to control the whole scope, and with Vite
  the `public/` directory is the dependency-free way to get a stable root URL.
  The M-1 worker is network-first; the cache-first app-shell worker lands in M5.
- Placeholder app icons are generated by `scripts/gen-icons.mjs` (zero deps).
